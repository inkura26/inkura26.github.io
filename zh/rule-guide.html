规则说明

更新日期：2026-03-04 · 本页由 Inkura 独立编写，仅用于产品兼容说明。

1. 适用范围

Inkura 支持社区兼容的规则格式，用于网页书源解析与 RSS 阅读。本说明描述的是 Inkura 的实现行为，不引用第三方项目教程文本。

2. 书源规则模型

  • 基础字段：书源地址、书源名称、启用状态。
  • 搜索规则：列表、书名、作者、详情地址、封面、简介。
  • 详情规则：元数据提取与目录地址。
  • 目录规则：章节列表、章节名、章节 URL、可选下一页。
  • 正文规则：正文提取与可选下一页规则。

3. 支持的表达类型

  • CSS 类选择器：适合 HTML 结构提取。
  • XPath：适合结构化节点定位。
  • JSONPath：适合 JSON 接口返回解析。
  • 正则替换：用于提取后的净化处理。
  • JavaScript：用于受限场景下的后处理。

4. 请求与 URL 选项

规则可包含请求选项，如 method、headers、charset、可选 web 渲染行为。仅在目标站点确有需要时启用。

5. RSS 规则

  • 订阅地址：必填，需返回合法 RSS/Atom XML。
  • 列表映射：标题、链接、发布时间、摘要。
  • 详情映射（可选）：从条目链接提取完整正文。
  • 刷新策略：按页码或时间戳，依据源站行为配置。

6. 合规提示

  • 仅配置你有合法访问权的内容来源。
  • 遵守目标站点条款、robots 规则与版权要求。
  • 避免账号滥用、绕过限制或被禁止的自动化抓取。

7. 常见排查

  • 列表解析失败：先校验选择器范围是否覆盖目标节点。
  • 正文为空：检查详情 URL 与正文规则执行顺序。
  • 乱码：在请求选项中显式指定 charset。

8. 联系方式

如需规则兼容支持，请联系 support@inkura.app。
